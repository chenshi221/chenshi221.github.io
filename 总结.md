## 发布策略

网站只发布经过筛选的公开笔记，内容位于 `content/`。

明确不发布：

- `组会`
- `AIMbench`
- `AIM-bench`
- `模板`
- `.obsidian`
- `.trash`
- `.claude`
- `.vscode`
- `CLAUDE.md`

这些目录或文件通过两层机制控制：

- `scripts/sync-notes.mjs`：同步时不会复制这些内容到 `content/`
- `quartz.config.ts` 的 `ignorePatterns`：构建时也会忽略这些路径

此外，`.gitignore` 也排除了这些不应提交的路径，防止误传到 GitHub。

## 常用命令

从 Obsidian vault 同步最新公开笔记：

```bash
cd /Users/chenshi/Desktop/blog1
pnpm sync:notes
```

本地构建：

```bash
pnpm quartz build
```

本地预览：

```bash
pnpm quartz build --serve --port 8080
```

发布更新：

```bash
git add -A
git commit -m "Update notes"
git push
```

push 后 GitHub Actions 会自动部署到 GitHub Pages。

## GitHub Pages 设置

- Pages source 已设置为 GitHub Actions workflow
- 部署工作流文件：`.github/workflows/deploy.yml`
- workflow 会执行：
  - `corepack enable`
  - `actions/setup-node` 使用 Node 22
  - `pnpm install --frozen-lockfile`
  - `pnpm quartz build`
  - `actions/deploy-pages`

`quartz.config.ts` 会在 GitHub Actions 中根据 `GITHUB_REPOSITORY` 自动推导 `baseUrl`。

## 隐私检查记录

首次发布前已检查：

- `组会`、`AIMbench`、`AIM-bench` 未进入本地 Git 历史
- 公开站点中 `AIMbench` 和 `组会` 页面返回 `404`
- 正常公开页面如 `知识点/DiT` 返回 `200`

后续继续发布前，建议再次检查：

```bash
git status --short --untracked-files=all
find content -path '*AIMbench*' -o -path '*AIM-bench*' -o -path '*组会*'
git diff --cached --name-only -- content/AIMbench content/AIM-bench content/组会
```

如果这些命令显示了不应发布的文件，应先停止提交和 push。
