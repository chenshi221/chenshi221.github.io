## Intro

### Motivation

前三代 Seedream 模型专注于文本到图像（T2I）生成的质量和文本渲染精度。然而，用户的需求远不止 T2I——图像编辑、多图生成（如漫画分镜、产品多视角）、图像理解和推理等同样是高频场景。

Seedream 4.0 的定位是：从一个优秀的 T2I 模型，进化为一个**多模态视觉生成系统**。同时，由于 3.0 的训练和推理成本较高，4.0 需要在不牺牲质量的前提下提升效率。

### 核心主张

1. **系统化能力拓展**：从 T2I 到编辑、多图生成、视觉推理的多任务统一
2. **高效 DiT**：通过架构优化（如混合注意力、层剪枝）降低计算成本
3. **高压缩 VAE**：更强的压缩比减少 token 数量，降低 DiT 的计算负担
4. **能力不退化**：在新增能力的同时，T2I 基础质量不降反升

### 贡献

1. 构建统一的多任务视觉生成系统（T2I + 编辑 + 多图）
2. 提出高效 DiT 架构变体，降低 30-50% 计算量
3. 设计高压缩 VAE（更高压缩比 + 更好重建质量）
4. 引入视觉推理能力（如"根据这张草图，生成成品图"）
5. 在效率和质量的 trade-off 上取得显著进展

## Method 核心方法

### 1. 多任务统一架构

Seedream 4.0 在单一模型中集成了多种视觉生成任务：

**Text-to-Image（T2I）**：继承 3.0 的 GA-ByT5 + DiT pipeline

**图像编辑**：
- 输入：源图像 + 编辑指令文本
- 类似 FLUX Kontext 的序列拼接方式，将源图像 VAE token、指令 token、噪声 token 拼接
- 支持指令式编辑（"把天空变成日落"）、inpainting、style transfer 等

**多图生成**：
- 同一 prompt 生成多张一致风格的图像
- 支持序列生成（如漫画分镜、故事板）
- 关键挑战：多图之间的一致性和连续性

**视觉推理生成**：
- 输入草图/简笔画 → 输出高质量成品图
- 输入设计稿 → 输出渲染图
- 需要模型具备"理解不完整输入并合理补全"的能力

### 2. 高效 DiT 架构

针对 DiT 计算量大的问题，4.0 进行了多项效率优化：

**混合注意力（Hybrid Attention）**：
- 全注意力只在部分层使用，其余层使用窗口注意力或线性注意力
- 在保持全局建模能力的同时显著减少计算量

**层剪枝与知识蒸馏**：
- 分析各层的表征贡献，移除冗余层
- 用大模型作为 teacher 蒸馏小模型

**动态 token 合并**：
- 在 DiT 的前几层合并语义相似的 token，减少后续层的计算量
- 在高分辨率图像上尤为有效

### 3. 高压缩 VAE

减少 VAE 输出的 token 数量是降低 DiT 计算量的最直接方法：

- **更高压缩比**：从 SD VAE 的 8x 压缩提升到 16x 或 32x
- **重建质量**：使用更强的解码器（更多层、更大容量）补偿压缩损失
- **对抗训练**：在 VAE 训练中加入 discriminator，提升重建的感知质量
- **两阶段解码**：VAE 输出低分辨率 latent → 超分网络输出最终图像

### 4. 训练策略升级

4.0 在训练上进行了多项优化：

- **课程学习**：先训练低分辨率基础 T2I，再逐步引入高分辨率、编辑、多图能力
- **能力不遗忘**：新增任务训练时混合旧任务数据，防止 catastrophic forgetting
- **数据配比优化**：T2I : 编辑 : 多图的比例通过实验确定

## 实验/评估/结果

![](https://arxiv.org/html/2509.20427v3/x1.png)

*Figure 2: Seedream 4.0 在 T2I 和图像编辑两个维度的整体评估——左侧为 T2I 结果对比（与 GPT-Image-1、Gemini-2.5 等），右侧为图像编辑结果对比。Seedream 4.0 在两个赛道均位列 Artificial Analysis Arena 榜首。*

### T2I 生成质量

- 在 GenEval、T2I-CompBench 等 benchmark 上与 Seedream 3.0 持平或略优
- 高效 DiT + 高压缩 VAE 的计算量降低约 40%，但质量无明显下降
- 中文文本渲染精度保持在 3.0 水平

### 图像编辑

- 指令式编辑的指令遵循度显著高于专用编辑模型的零样本表现
- 非编辑区域保持度良好
- 多轮编辑能力稳定（可连续编辑 3-5 轮不退化）

![](https://arxiv.org/html/2509.20427v3/x4.png)

*Figure 3: Seedream 4.0 单图编辑对比示例——与 GPT-Image-1、Gemini-2.5 在同一 prompt 下的编辑结果对比。Seedream 4.0 在指令遵循和图像一致性之间取得了更好的平衡，支持风格转换、文本翻译、视角变换等多种编辑类型。*

### 多图生成

- 多图一致性：图与图之间的风格、角色、场景高度一致
- 支持多图序列生成（如 4 格漫画）
- 多视角生成：同一物体的不同角度保持几何一致性

### 效率 Benchmark

- **高效 DiT**：FLOPs 降低 35%，推理延迟降低 30%
- **高压缩 VAE**：token 数量减少 4x，DiT 输入序列长度大幅缩短
- **总体**：同等质量下推理成本降低约 40-50%

---

## 结论

Seedream 4.0 完成了从单任务 T2I 模型到多模态视觉生成系统的关键跃迁。通过高效 DiT 架构和高压缩 VAE，在新增编辑、多图、推理能力的同时显著降本增效，为工业化部署奠定了基础。

---

## 思考

### 优点

1. **系统化思维**：4.0 不是简单地在 3.0 上加功能，而是从系统架构层面重新设计——高效 DiT + 高压缩 VAE + 多任务训练，三个层面的优化相互配合。

2. **实际部署导向**：效率优化（40-50% 成本降低）对实际产品（豆包）的运营至关重要。论文在"学术指标"和"工程指标"之间取得了难得的平衡。

3. **增量能力拓展的策略正确**：通过课程学习和数据配比控制了新增能力对基础 T2I 的干扰，避免了"加功能、掉质量"的常见陷阱。

4. **高压缩 VAE 的技术价值**：这实际上是独立于 Seedream 的贡献——更高压缩比的 VAE 可以造福整个扩散模型社区。

### 缺点与待解决问题

1. **多图生成的评估不足**：多图一致性缺少标准化的 benchmark 评估，主要依赖定性展示。多帧之间的长期一致性（如 10 帧以上）可能存在问题。

2. **视觉推理的边界不清晰**：草图到成品的能力在什么条件下有效？什么复杂的草图会导致失败？论文没有系统分析。

3. **高效 DiT 的消融不够细**：混合注意力、层剪枝、token 合并各自贡献了多少？是否有更优的组合？缺少逐步消融。

4. **VAE 的压缩极限**：16x-32x 压缩是否已接近 VAE 的信息瓶颈？更高压缩比的可行性未讨论。
